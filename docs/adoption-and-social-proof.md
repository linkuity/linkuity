# Adoption and social proof — placeholder strategy

The README deliberately contains **no** "trusted by" logos, customer names,
record-processing totals, download counts, performance numbers, or testimonials.
Linkuity is early; adding social proof before it is real would mislead evaluators and
erode the credibility the project is trying to build.

This document is the plan for adding social proof **later, once it genuinely exists** —
so that when the time comes it is added honestly and consistently, not invented.

## What we will not do

- Invent customers, adoption figures, or usage totals.
- Publish performance or accuracy numbers before they are reproducible (see
  [benchmarks.md](benchmarks.md)).
- Add stars/downloads badges as a substitute for substance.

## What we will add, when it is real

| Signal | Trigger | Where it goes |
|--------|---------|---------------|
| GitHub stars / contributor count | Organic growth | Existing badges only; no editorializing. |
| Reproducible benchmark results | The [benchmark plan](benchmarks.md) is executed | A results section that links the reproducibility bundle. |
| Case studies / testimonials | A real user agrees to be named | A dedicated "Who uses Linkuity" doc, with permission and a link. |
| Integrations / ecosystem | A real, maintained integration exists | Documentation for that integration. |
| Download / package stats | Packaged releases exist and have real usage | Release notes / a stats badge from the registry itself. |

## Principle

Every claim in public-facing docs must be verifiable from the repository or from a named,
permissioned source. When in doubt, leave it out.
